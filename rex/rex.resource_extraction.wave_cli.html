rex.resource_extraction.wave_cli module¶

WaveX¶

WaveX Command Line Interface

WaveX [OPTIONS] COMMAND [ARGS]...

Options

-h5, --wave_h5 <wave_h5>¶

Required Path to Resource .h5 file

-o, --out_dir <out_dir>¶

Required Directory to dump output files

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

multi-site¶

Extract multiple sites given in ‘–sites’ .csv or .json as “latitude”, “longitude” pairs OR “gid”s

WaveX multi-site [OPTIONS] COMMAND [ARGS]...

Options

-s, --sites <sites>¶

Required .csv or .json file with columns “latitude”, “longitude” OR “gid”

dataset¶

Extract given dataset for all sites

WaveX multi-site dataset [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

sam¶

Extract SAM variables

WaveX multi-site sam [OPTIONS]

region¶

Extract a single dataset for all pixels in the given region

WaveX region [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

-r, --region <region>¶

Required Region to extract

-col, --region_col <region_col>¶

Meta column to search for region

sam-file¶

Extract all datasets needed for SAM for the nearest pixel to the given (lat, lon) coordinates OR the given resource gid

WaveX sam-file [OPTIONS]

Options

-ll, --lat_lon <lat_lon>¶

(lat, lon) coordinates of interest

-g, --gid <gid>¶

Resource gid of interest

site¶

Extract a single dataset for the nearest pixel to the given (lat, lon) coordinates OR the given resource gid

WaveX site [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

-ll, --lat_lon <lat_lon>¶

(lat, lon) coordinates of interest

-g, --gid <gid>¶

Resource gid of interest

timestep¶

Extract a single dataset for a single timestep Extract only pixels in region if given.

WaveX timestep [OPTIONS]

Options

-ts, --timestep <timestep>¶

Required Timestep to extract

-d, --dataset <dataset>¶

Required Dataset to extract

-r, --region <region>¶

Region to extract

-col, --region_col <region_col>¶

Meta column to search for region
