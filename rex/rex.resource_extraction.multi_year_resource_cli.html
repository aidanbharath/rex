rex.resource_extraction.multi_year_resource_cli module¶

MultiYearX¶

ResourceX Command Line Interface

MultiYearX [OPTIONS] COMMAND [ARGS]...

Options

-h5, --resource_path <resource_path>¶

Required Path to Resource .h5 files

-o, --out_dir <out_dir>¶

Required Directory to dump output files

-yrs, --years <years>¶

List of years to access, by default None

-hsds, --hsds¶

Boolean flag to use h5pyd to handle .h5 ‘files’ hosted on AWS behind HSDS

-res, --res_cls <res_cls>¶

Resource type

Options

Resource|NSRDB|Wind|Wave

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

multi-site¶

Extract multiple sites given in ‘–sites’ .csv or .json as “latitude”, “longitude” pairs OR “gid”s

MultiYearX multi-site [OPTIONS] COMMAND [ARGS]...

Options

-s, --sites <sites>¶

Required .csv or .json file with columns “latitude”, “longitude” OR “gid”

dataset¶

Extract given dataset for all sites

MultiYearX multi-site dataset [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

region¶

Extract a single dataset for all pixels in the given region

MultiYearX region [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

-r, --region <region>¶

Required Region to extract

-col, --region_col <region_col>¶

Meta column to search for region

site¶

Extract a single dataset for the nearest pixel to the given (lat, lon) coordinates OR the given resource gid

MultiYearX site [OPTIONS]

Options

-d, --dataset <dataset>¶

Required Dataset to extract

-ll, --lat_lon <lat_lon>¶

(lat, lon) coordinates of interest

-g, --gid <gid>¶

Resource gid of interest
